    Часть 1: Создать классификатор дерева решений¶

    В этой части лабораторной работы вы создадите классификатор дерева решений, который будет обучаться благодаря маркированному набору данных.

    Набор данных содержит имена и демографические данные для каждого пассажира. Кроме того, включены детали поездки каждого пассажира. Из этих данных мы можем построить дерево решений, которое иллюстрирует факторы способствовавшие положительному или отрицательному исходу для пассажира рейса.

    Наборы данных содержат следующие переменные:

    Переменная Описание
    1. PassengerID Уникальный идентификатор для каждого пассажира
    2. Survival Выжил ли пассажир? (0 = Нет; 1 = Да)
    3. Pclass Класс пассажирских билетов. (1 = Первый класс; 2 = Второй класс; 3 = Третий класс)
    4. Name ФИО пассажира (фамилия, имя)
    5. Gender Мужской или женский
    6. Age Возраст в годах. В основном целые числа и числа с плавающей запятой для детей младше одного года.
    7. SibSp Число братьев и сестер на борту.
    8. Parch Число родителей или детей на борту.
    9. Ticket Номер билета
    10. Fare Сумма, уплаченная за проезд в британских фунтах 1970 года
    11. Cabin Номер каюты
    12. Embarked Порт высадки (C = Шербур; Q = Куинстаун; S = Саутгемптон)

    e) Интерпретируем дерево¶

    Из дерева мы можем увидеть несколько вещей: во-первых, в корне дерева есть переменная Gender, что указывает на то, что это самый важный фактор в классификации. На третьем уровне, в крайнем правом углу, мы видим, что 415 человек были мужчинами и платили за проезд менее 26,2686. Наконец, самый верхний уровень этого промежуточного узла указывает, что 15 из этих пассажиров были ниже возраста 13.5, а остальные 400 были старше этого возраста.

    Наконец, элементы в массиве значений указывают на выживаемость. Первое значение - это количество умерших людей, а второе - количество выживших для каждого критерия. Корневой узел сообщает нам, что из нашего образца 549 ​​человек погибло, а выжило 342.

    Энтропия - это мера шума в решении. Шум можно рассматривать как неопределенность. Например, в узлах, в которых решение приводит к равным значениям в массиве значений выживаемости, энтропия имеет наивысшую возможную величину, равную 1.0. Это означает, что модель не смогла окончательно сделать классификационное решение на основе входных переменных. Для значений с очень низкой энтропией решение было гораздо более четким, а разница в числе выживших и жертв намного выше.

    Шаг 2. Создайте входные и выходные переменные для данных обучения и тестирования¶

    Библиотека sklearn включает в себя модули, которые помогают с выбором модели. Мы будем импортировать из sklearn.model_selection метода train_test_split(). Этот метод автоматически разбивает весь набор данных, возвращая в общей сложности четыре массива numpy, два для характеристик (тестирование и проверка) и два для меток (тестирование и проверка). Один параметр метода определяет долю наблюдений, используемых для тестирования и обучения. Другой параметр указывает начальное значение, которое будет использоваться для случайного назначения наблюдения на тестирование или обучение. Это используется для того, чтобы другой пользователь мог повторить вашу работу, получая те же самые назначения наблюдений наборам данных. Синтаксис метода:

    ```train_test_split(input_X, target_y, test_size=0.4, random_state=0)```

    40% данных будут использоваться для тестирования. Случайное начальное значение устанавливается равным 0.

    Метод возвращает четыре значения. Они возвращаются в следующем порядке: входные переменные для данных обучения, входные переменные для тестируемых данных, выходные переменные для данных обучения и выходные переменные для тестируемых данных.

    Часть 4 Для дальнейшего изучения (необязательно)¶

    Если у вас есть время и вы заинтересованы, вы можете попробовать следующее и посмотреть, как это влияет на дерево решений.

    1. Удалите наблюдения с отсутствующими значениями Age.¶

    Использование среднего значения для замены отсутствующих значений возраста может повлиять на точность модели. Одним из подходов к этому может быть удаление всех наблюдений с отсутствующими возрастными значениями. Хотя это уменьшит размер набора учебных данных, это может повысить точность.

    2. Удалите входные переменные.¶

    Еще одна проблема с этим типом анализа - это определение того, какие входные переменные или характеристики важны для точности классификатора. Один из способов сделать это - попробовать запустить классификатор с различными наборами входных переменных, отредактировав список переменных, которые соответствуют модели.
